✅ 1. UserCredential
   Location: app/models/models.py
   Purpose: User authentication and account management
   Relationships: creator_profile (1:1), user_profile (1:1), user_events (1:N), user_interests (1:1)
   Key Fields: id, username, email, password, created_at, updated_at, is_active
   Constraints: Email unique, Username unique, Password required
   Indexes: id (PK), email (UNIQUE), username (UNIQUE), created_at

✅ 2. CreatorProfile
   Location: app/models/models.py
   Purpose: Event creator and organizer profiles
   Relationships: user (1:1 FK), events (1:N)
   Key Fields: id, user_id, name, phoneno, address, brandname, email, bio, profile_picture
   Constraints: user_id unique, email unique
   Indexes: user_id (FK), email (UNIQUE), brandname

✅ 3. UserProfile
   Location: app/models/models.py
   Purpose: Regular user/attendee profiles
   Relationships: user (1:1 FK)
   Key Fields: id, user_id, name, phoneno, address, email, profile_picture
   Constraints: user_id unique, email unique
   Indexes: user_id (FK), email (UNIQUE)

✅ 4. EventCreation
   Location: app/models/models.py
   Purpose: Event details, pricing, and payment information
   Relationships: creator (1:N FK), user_events (1:N), table_categories (1:N)
   Key Fields: id, creator_id, event_name, date, category, price, vip_price, vvip_price, vvvip_price, table_price
   Pricing Support: 4 tiers + bank information (account_name, account_number, bank)
   Constraints: Prices >= 0, event_name required, category required
   Indexes: name, category, date (FULLTEXT on name/summary)

✅ 5. TableCategory
   Location: app/models/models.py
   Purpose: Table seating arrangements for events
   Relationships: event (1:N FK)
   Key Fields: id, event_id, name, capacity, price, available_tables
   Constraints: capacity > 0, price >= 0, available_tables >= 0
   Indexes: event_id (FK), name

✅ 6. UserEvent
   Location: app/models/models.py
   Purpose: Ticket purchases and attendance records
   Relationships: user (1:N FK), event (1:N FK)
   Key Fields: id, user_id, event_id, token, ticket_type, isVerified, verified_at
   Constraints: token unique, user_id required, event_id required
   Indexes: user_id (FK), event_id (FK), token (UNIQUE)

✅ 7. UserInterest
   Location: app/models/models.py
   Purpose: User interest preferences for recommendations
   Relationships: user (1:1 FK)
   Key Fields: id, user_id, interests (JSON array)
   Storage: JSON for flexible category storage
   Constraints: user_id unique, user_id required
   Indexes: user_id (FK, UNIQUE)

       EventCreation, TableCategory, UserEvent, UserInterest

   events = user.user_events
